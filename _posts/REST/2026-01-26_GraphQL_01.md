#### Chap 01. GraphQL
- API 를 만들 때 사용할 수 있는 쿼리 언어
- GraphQL은 선언형(declarative) 데이터 페칭(fetching) 언어라고 일컬어진다.
  - 무슨 데이터가 필요한지 요구사항만 작성하면 어떻게 가져올 지 신경쓰지 않아도 된다.
- GraphQL 명세
  - GraphQL은 클라이언트와 서버간의 통신 명세(스펙)
- GraphQL 설계 원칙
  - 위계적: GraphQL 쿼리는 위계성을 띠고 있다. 필드 안에 다른 필드가 중첩될 수 있으며, 반환 데이터는 형태가 서로 같다
  - 제품 중심적: 클라이언트가 요구하는 데이터와 지원하는 언어 및 런타임에 맞춰 동작
  - 엄격한 타입 제한: GraphQL 서버는 GraphQL 타입 시스템을 사용
  - 클라이언트 맞춤 쿼리: GraphQL 서버는 클라이언트 쪽에서 받아 사용할 수 있는 데이터 제공
  - 인트로스펙티브(introspective): GraphQL 서버가 사용하는 타입 시스템에 대한 쿼리를 작성할 수 있다.
- REST 의 단점
  - over fetching: 필요하지 않은 데이터를 너무 많이 받아 온다.
  - under fetching: 영화(movies/{movieId})를 조회 -> 영화에 나오는 출연진을 조회하려면(characters/{characterId})를 추가적으로 요청해야 한다
  - rest endpoint 관리: 유연성이 부족하여 요청을 최소화하고자 커스텀 엔드포인트를 생성해야 한다. /character-with-movie-title
- GraphQL 사용
  - API를 효율적으로 개발하기 위해 사용
  - GraphQL Client: 개발자가 빠르게 작업할 수 있는 환경을 만들어주고 애플리케이션의 성능과 효율성을 올려준다.

#### Chap 03. GraphQL 쿼리어
- SQL과 GraphQL의 QL 은 Query Language 로 쿼리 언어이지만 SQL은 DB 대상, GraphQL은 API 로 보내는 차이가 있다
  - GraphQL은 저장 환경을 가리지 않는다. DB, File System, REST API, WebSocket, 다른 GraphQL API 등
  - select: query // insert, update, delete: mutation 이라는 데이터 타입을 가지고 데이터를 조작한다.
  - subscription 타입을 사용하면 소켓 연결로 전달되는 데이터 변경 사항을 감지할 수 있다.

##### 3.2 GraphQL 쿼리
- 쿼리 작업으로 API에 데이터 요청을 할 수 있으며 쿼리 안에 GraphQL 서버에서 받고 싶은 데이터를 작성한다.
  ```GraphQL
  query {
    allLifts {
      name
      status
    }
  }
  ```
  - 이렇게 요청하면 JSON 응답에 allLifts 배열과 각각의 리프트에 대한 name, status가 들어 있게 된다.
  ````json
  {
    'allLifts': [{name: "", status: ""}, ... ]
  }
  ```
- 두개의 문서를 한번에 요청할 수도 있으며 쿼리 한번에 여러 종류의 데이터를 모두 받을 수 있다.
  - 루트 타입: Query와 같은 타입을 일컬으며 하나의 작업을 수행
  - 셀렉션 세트: 필요한 필드를 중괄호로 감싼 것을 말하며 셀렉션 세트는 서로 중첩할 수 있다.
  - 쿼리 인자: 쿼리 결과에 대한 필터링
  ```GraphQL
  query lifts {
    allLifts {
      name
      status
    }
  }
  query trails {
    allTrails {
      name
      difficulty
    }
  }
  ```
  ```GraphQL
  query closedLifts {
    allLifts(status: CLOSED) {
      name
      status
    }
  }
  ```
- 엣지와 연결
  - GraphQL 쿼리어에서 필드는 스칼라 타입과 객체 타입에 속하게 된다
    - 스칼라 타입: 원시 타입과 비슷하며 Int, Float, String, Boolean, ID(고유 식별자) 총 5개 존재
    - 객체 타입: 스키마에 정의한 필드를 그룹으로 묶어 둔 것
    ```GraphQL
    query trailsAccessedByJazzCat {
      Lift(id: "jazz-cat") {
        capacity
        trailAccess {
          name
          difficulty
        }
      }
    }
    ```
    - capacity는 스칼라 타입, trailAccess는 Trail 타입(객체 타입)
- 프래그먼트
  - 셀력션 세트의 일종으로 여러번 재 사용할 수 있다.
  ```GraphQL
  fragment liftInfo on Lift {
    name
    status
    capacity
  }
  query {
    Lift(id: "jazz-cat") {
      ...liftInfo
      trailAccess {
        name
        difficulty
      }
    }
    Trail(id: "river-run") {
      name
      difficulty
      accessedByLifts {
        ...liftInfo
      }
    }
  }
  ```
  - 유니언 타입
    - 타입 여러개를 한번에 리스트에 담아 반환하고 싶다면 유니언 타입을 만들면 된다
    - 인라인 프래그먼트: 쿼리 안에서 특정 타입을 바로 셀렉션 세트에 넣어버릴 수 있다.
      ```GraphQL
      query schedule {
        agenda {
          ...on Workout {
            name
            reps
          }
          ...on StudyGroup {
            name
            subjects
            students
          }
        }
      }
      ```
    - 유니언 타입 쿼리
      ```GraphQL
      query today {
        agenda {
          ...workout
          ...study
        }
      }
      fragment workout on Workout {
        name
        reps
      }
      fragment study on Study {
        name
        subjects
        students
      }
      ```
    - 인터페이스
      - 필드 하나로 객체 타입을 여러개 반환할 때 사용하며 추상 타입으로 유사한 객체 타입을 만들 때 구현해야할 필드를 모아둔 것
##### 3.2 뮤테이션
- 뮤테이션은 데이터를 새로 쓰거나 수정할 때 사용하며 쿼리 작성 방법은 비슷하다.
- 이름을 붙여야 하며 객체 타입이나 스칼라 타입의 반환값을 가지는 셀렉션 세트가 들어간다.
  ```GraphQL
  mutation burnItDown {
    deleteAlLData // 데이터 전체 삭제
  }
  ```
- 데이터 입력 및 반환 값 정의
```GraphQL
mutation createSong {
  addSong(title:"No Scrubs", numberOne: true, performerName: "TLC") {
    id
    title
    numberOne
  }
}
```
- 데이터 수정: id가 jazz-cat인 것에 status를 닫힘으로 변경
```GraphQL
mutation closeLift {
  setLiftStatus(id: "jazz-cat", status: "CLOSED") {
    name
    status
  }
}
```
- 쿼리 변수 사용하기
```GraphQL
mutation createSong($title:String! $numberOne:Int $by:String!) {
  addSong(title:$title, numberOne:$numberOne, performerName:$by) {
    id
    title
    numberOne
  }
}
```
